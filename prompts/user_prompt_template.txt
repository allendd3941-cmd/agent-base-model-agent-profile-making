請生成 100 個用於 agent-based traffic simulation 的 agent profile。

模擬情境：
台南市亞太國際棒球訓練中心在球賽進場與散場尖峰時段，周邊道路、人潮、停車、接駁與一般通勤活動會產生短期交通衝擊。

生成目的：
這些 agent 會被後續語言模型讀取，用來推論旅次目的、出發時間、交通工具選擇、路線選擇、停車行為、是否避開壅塞、是否提前或延後出發等行為。

角色組成要求：
每個 agent 必須具備差異性，至少包含以下類型，但不要新增 agent_type 欄位，只能透過 identity 與 traits 的自然語言內容表達：
1. 去亞太棒球場看球的觀眾
2. 非觀賽的一般通勤者
3. 球場周邊居民
4. 接送家人或朋友的人
5. 場館或活動工作人員
6. 球場周邊商家、外送員、計程車司機或其他現實中可能受影響的人

內容要求：
1. 所有欄位 value 都必須是繁體中文自然語言。
2. 可以使用數字與單位，但不能只有代碼、縮寫或單一數字。
3. 每位 agent 的年齡、職業、收入、家庭收入、交通工具、居住地必須符合現實邏輯。
4. household_income 不得明顯低於該 agent 的個人 wage，除非 wage 明確表示無固定收入或兼職收入。
5. vehicle_ownership 需要描述是否有汽車、機車、自行車，或是否依賴公車、火車、接駁、計程車、步行等。
6. traits 中每個 array 必須包含 2 到 4 個自然語言句子或片語。
7. short_term_memory 與 long_term_memory 必須固定輸出空 array：[]。
8. 不要新增、刪除或重新命名任何欄位。

輸出格式要求：
1. 只能輸出合法 JSON。
2. 最外層必須是 JSON array。
3. array 中必須有剛好達要求數量的 agent object。
4. 第一個字元必須是 [，最後一個字元必須是 ]。
5. 不要輸出 markdown。
6. 不要輸出 ```json。
7. 不要輸出任何說明、註解、前言或結語。
8. JSON object 的 key 必須使用雙引號。
9. 不得出現 trailing comma。
10. 你需要自我檢查是否符合正確的json結構，且沒有缺漏、不正確的排列等等，再給我回覆。
