## 核心要求

1. 僅根據我提供的「回傳環境內容」整理，不得自行發明不存在的資料。
2. 目前 GAMA 已只送出精簡 payload；請只整理下方列出的代表欄位，不要還原、推測或展開已刪減的 raw road、raw location、API 狀態等欄位。
3. 每個 agent 必須各自形成一份 RAG document，並且要能用 `agent_id` 對應。
4. 整體環境狀況必須形成一份獨立 RAG document。
5. `memory` 只保留精簡旅行記憶欄位；不得要求完整原始 travel memory。
6. 可以新增 `summary_text`、`interpretation`、`metadata`、`keywords` 等方便檢索的欄位，但不能取代精簡原始資料。
7. 輸出必須是合法 JSON，不要輸出 Markdown，不要加解釋文字，不要包 ```json。
8. 若某欄位在 payload 中不存在，請在 `coverage_check.missing_fields` 中列出，不要自行補值。
9. 若某欄位存在但值為空字串、null 或空陣列，仍要保留。

## 必須理解的 payload 結構

最外層欄位：
- `model`: GAMA 模型名稱。
- `cycle`: 目前模擬 cycle，也就是第幾個模擬 step。
- `environment`: 整體環境摘要。
- `agents_status`: 所有 agent 的精簡狀態清單。

`environment` 欄位：
- `cycle`
- `elapsed_minutes`
- `agent_count`
- `destination_town`
- `active_road_count`
- `crowded_road_count`
- `average_congestion_proxy`

`agents_status` 每個元素欄位：
- `agent_id`
- `origin_town`
- `destination_town`
- `active_mode`
- `environment`
- `memory`

`agent.environment` 欄位：
- `current_town`
- `current_road_id`
- `route_status`
- `nearby_agent_count`
- `congestion_proxy`
- `distance_to_destination_m`

`memory` 每筆欄位：
- `cycle`
- `current_town`
- `current_road_id`
- `active_mode`
- `route_status`
- `nearby_agent_count`
- `congestion_proxy`
- `distance_to_destination_m`

如果 payload 中仍出現 `roads_flow`、`road_schema`、`town_schema`、`available_towns` 或 `environment_variable_catalog`，不要逐筆展開；只有在有助於 metadata 或 coverage_check 時才引用。

## 整理規則

- `documents` 第一筆必須是整體 `environment_snapshot`。
- 後續每一筆 document 對應一個 `agents_status` agent。
- agent document 必須以 `agent_id` 作為主要識別。
- `summary_text` 要適合自然語言檢索，例如能被查詢「誰在東區」、「誰接近目的地」、「目前是否壅塞」時命中。
- `content` 中只保留精簡欄位，不要輸出未列入 payload 的欄位。
- `interpretation` 只能根據欄位意義解釋，不得新增 payload 沒有的事實。
- 如果 `memory` 為空，仍要輸出空陣列並在 `summary_text` 說明該 agent 尚無旅行記憶。
- 如果 `route_status` 是 `arrived`，必須在 `summary_text` 中說明該 agent 已抵達。
- 如果 `route_status` 是 `waiting_for_api_origin`，必須說明該 agent 尚未完成生成地套用。
- 不要回傳 active mode 決策；本任務只負責整理 RAG 原始資料。
