## 核心要求

1. 僅根據我提供的「回傳環境內容」整理，不得自行發明不存在的資料。
2. 不得遺漏 payload 中任何欄位。
3. 必須保留原始欄位名稱與原始值，包含有空格的欄位，例如 `agent name`、`active mode`。
4. 每個 agent 必須各自形成一份 RAG document，並且要能用 `agent name` 對應。
5. 整體環境狀況必須形成一份獨立 RAG document。
6. 所有 `travel_memory` 必須完整保留，不能只摘要。
7. 可以新增 `summary_text`、`interpretation`、`metadata`、`keywords` 等方便檢索的欄位，但不能取代原始資料。
8. 輸出必須是合法 JSON，不要輸出 Markdown，不要加解釋文字，不要包 ```json。
9. 若某欄位在 payload 中不存在，請在 `coverage_check.missing_fields` 中列出，不要自行補值。
10. 若某欄位存在但值為空字串、null 或空陣列，仍要保留。

## 必須理解的 payload 結構

最外層欄位：
- `model`: GAMA 模型名稱。
- `cycle`: 目前模擬 cycle，也就是第幾個模擬 step。
- `agents`: 所有 agent 清單。
- `environment`: 當前整體環境狀況。

`agents` 每個元素欄位：
- `agent name`: agent 名稱，必須用於 document metadata 與 response 對應。
- `travel_memory`: 該 agent 的旅行記憶清單。
- `current_state`: 該 agent 當前狀態摘要。

`travel_memory` 每筆欄位：
- `cycle`
- `elapsed_minutes`
- `current_location`
- `current_town`
- `current_road_id`
- `current_road_name`
- `current_road_type`
- `active_mode`
- `selected_action`
- `speed_used_kmh`
- `distance_moved_m`
- `distance_to_destination_m`
- `nearby_agent_count`
- `api_request_status`
- `api_response_summary`
- `warning`

`current_state` 欄位：
- `residential_location`
- `current_town`
- `current_road_id`
- `destination_town`
- `active mode`
- `speed_kmh`
- `route_status`
- `nearby_agent_count`

`environment.simulation` 欄位：
- `cycle`
- `step_minutes`
- `elapsed_minutes`
- `max_steps`
- `agent_count`

`environment.spatial` 欄位：
- `destination_town`
- `available_towns`
- `crs_note`

`environment.traffic` 欄位：
- `active_road_count`
- `crowded_road_count`
- `average_congestion_proxy`
- `roads_with_flow`

`roads_with_flow` 每條道路欄位：
- `road_id`
- `NAME`
- `highway`
- `highway_ty`
- `length`
- `lanes`
- `capacity`
- `speed_car`
- `speed_moto`
- `time_car`
- `time_moto`
- `time`
- `current_flow`
- `congestion_proxy`

## 整理規則

- `documents` 第一筆必須是整體 `environment_snapshot`。
- 後續每一筆 document 對應一個 agent。
- agent document 必須以 `agent name` 作為主要識別。
- `summary_text` 要適合自然語言檢索，例如能被查詢「誰在東區」、「誰接近安南區」、「哪些道路壅塞」時命中。
- `content.raw` 或等價位置必須完整保留原始欄位與原始值。
- `interpretation` 只能根據欄位意義解釋，不得新增 payload 沒有的事實。
- 如果 `roads_with_flow` 為空，仍要輸出空陣列並在 `summary_text` 說明目前沒有道路流量資料。
- 如果 `travel_memory` 為空，仍要輸出空陣列並在 `summary_text` 說明該 agent 尚無旅行記憶。
- 如果某 agent 有 warning，必須在 summary_text 中提到。
- 如果 `route_status` 是 `arrived`，必須在 summary_text 中說明該 agent 已抵達。
- 如果 `route_status` 是 `waiting_for_api_origin`，必須說明該 agent 尚未完成生成地套用。
- 不要回傳 active mode 決策；本任務只負責整理 RAG 原始資料。
