# API Reference

## `POST /from-gama`

Receives simulation state from GAMA and returns a decision result generated by the LLM pipeline.

```text
POST http://127.0.0.1:8000/from-gama
Content-Type: application/json
```

## Supported Payload Patterns

The server accepts extra fields to remain compatible with GAMA payloads.

### Initialization Payload

Used when GAMA initializes agents.

```json
{
  "request_type": "init_agents",
  "model": "TrafficABM_Tainan_LLM",
  "model_name": "TrafficABM_Tainan_LLM",
  "cycle": 0,
  "vehicles": 10,
  "requested_agents": [
    {
      "agent_id": "vehicle_001",
      "fallback_origin_town": "佳里區",
      "fixed_destination_town": "安南區",
      "active_mode": {}
    }
  ]
}
```

### Step Update Payload

Used during simulation cycles.

```json
{
  "request_type": "step_update",
  "model": "TrafficABM_Tainan_LLM",
  "model_name": "TrafficABM_Tainan_LLM",
  "cycle": 1,
  "step_index": 1,
  "elapsed_minutes": 5,
  "agents_status": [
    {
      "agent_id": "vehicle_001",
      "origin_town": "佳里區",
      "destination_town": "安南區",
      "memory": []
    }
  ],
  "roads_flow": []
}
```

### Compatibility Payload

The server also accepts the earlier `agents` format.

```json
{
  "model": "TrafficABM_Tainan_LLM",
  "cycle": 1,
  "agents": [
    {
      "agent name": "agent_001",
      "travel_memory": [],
      "current_state": {}
    }
  ],
  "environment": {}
}
```

## Response

The current endpoint returns the raw decision-making output generated by the LLM pipeline. In the current research prototype, the response format depends on the prompt and model behavior.

Future versions should validate the LLM response against a strict JSON schema before returning it to GAMA.

## Local Run Command

```bash
uvicorn server:app --host 127.0.0.1 --port 8000 --reload
```

## Dependencies

This endpoint requires:

- FastAPI server running locally.
- Ollama endpoint configured through `.env`.
- Prompt files under `prompts/`.
- Writable local `output/` directory for generated artifacts.
